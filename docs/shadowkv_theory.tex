\section{Setup}

Let $\mathcal{P}$ denote the set of observed token prefixes tracked by the system.
For each prefix $p \in \mathcal{P}$ of length $L = |p|$, the system maintains:

\begin{itemize}
    \item $\hat{f}(p)$ is an exponential moving average (EMA) estimate of reuse frequency, updated as
    \[
        \hat{f}_t(p) = \alpha + (1 - \alpha)\,\hat{f}_{t-1}(p), \quad \alpha \in (0,1)
    \]
    where the update fires when a request whose token sequence begins with $p$ is observed.

    \item $n(p)$ is the raw observation count for prefix $p$.
\end{itemize}

\begin{definition}[Reuse probability estimate]
Define the confidence-adjusted reuse probability as
\[
    \hat{P}(p) = \hat{f}(p) \cdot \gamma(p), \qquad
    \gamma(p) = \min\!\left(\frac{n(p)}{n_{\min} + 2},\; 1\right)
\]
where $n_{\min}$ is the minimum observation threshold.
The factor $\gamma(p)$ shrinks the estimate toward zero when $p$ has been seen few times,
penalising overconfident speculation on rare prefixes.
\end{definition}

\section{Expected Benefit}

When prefix $p$ is cached and a matching request arrives, the system avoids recomputing
$L$ attention key-value pairs. Let $\beta > 0$ (ms per token) be the prefill cost per token.

\begin{definition}[Expected benefit]
\[
    B(p) = \hat{P}(p) \cdot \min(L,\, L_{\max}) \cdot \beta
\]
where $L_{\max}$ caps the prefix length beyond which marginal savings are diminished
by memory pressure effects.
\end{definition}

\section{Expected Cost}

Speculatively precomputing prefix $p$ during an idle window incurs two costs:

\begin{enumerate}
    \item A fixed overhead $c_0 > 0$ (ms) for scheduling and kernel launch.
    \item A length-proportional compute cost and a memory pressure penalty.
\end{enumerate}

Let $m(p)$ (MB) denote the memory footprint of the KV state for $p$,
and let $\rho \in [0,1]$ be the current memory utilisation ratio of the cache bank.

\begin{definition}[Expected cost]
\[
    C(p) = \left(c_0 + m(p)\cdot\delta + \max(L - L^*,\, 0)\cdot\lambda\right)
    \cdot (1 + \mu\,\rho)
\]
where:
\begin{itemize}
    \item $\delta > 0$ is the memory penalty per MB
    \item $\lambda > 0$ is the per-token length penalty beyond the preferred length $L^*$
    \item $\mu > 0$ is the pressure multiplier scaling cost under memory contention
\end{itemize}
The factor $(1 + \mu\rho)$ reflects that precomputing under high memory pressure
is more expensive because it is likely to trigger evictions.
\end{definition}

\section{Admission Rule}

\begin{definition}[Net value and admission criterion]
Define the expected net value of speculatively admitting prefix $p$ as
\[
    V(p) = B(p) - C(p)
\]
The admission policy admits $p$ if and only if all three conditions hold:
\begin{align}
    \hat{f}(p) &\geq f_{\min} \tag{C1: frequency gate} \\
    V(p) &\geq \tau \tag{C2: net value floor} \\
    \frac{B(p)}{C(p)} &\geq \eta \tag{C3: benefit-cost ratio}
\end{align}
for thresholds $f_{\min} > 0$, $\tau > 0$, and $\eta > 1$.
\end{definition}

Condition (C1) is a prior filter that blocks prefixes without sufficient frequency evidence.
Condition (C2) ensures the absolute net gain exceeds a minimum useful threshold $\tau$.
Condition (C3) provides a relative safety margin: even if $V(p) > \tau$,
a low ratio $B/C$ indicates that a small overestimate of reuse probability
would make the decision unprofitable.

\section{Ranking and Budget}

Given a slack window of duration $W$ ms and a per-idle admission budget $K$,
the policy selects at most $K$ prefixes from the admitted set.
Candidates are ranked by the score

\[
    s(p) = \frac{V(p)}{C(p)} + \epsilon_1 \min(L, L^*) + \epsilon_2 \min(n(p), n_{\max})
\]

where $\epsilon_1, \epsilon_2 \geq 0$ are small tie-breaking weights that
favour longer and better-observed prefixes among candidates of similar value efficiency.
The leading term $V(p)/C(p)$ is a return-on-investment ratio:
it selects prefixes that yield the most net value per unit of speculative compute spent.

\begin{proposition}[Waste bound]
Let $\phi = \Pr[\text{admitted prefix is never reused}]$ be the miss rate of the speculation policy.
For any admitted prefix $p$, the expected wasted compute is at most
\[
    \mathbb{E}[\text{waste}(p)] \leq \phi \cdot C(p)
\]
Since the admission rule requires $B(p) \geq \eta \cdot C(p)$ with $\eta > 1$,
a policy that achieves $\phi < 1/\eta$ has positive expected net value in expectation.
\end{proposition}

\noindent
This gives a direct interpretation of the ratio threshold $\eta$:
it encodes the maximum tolerable speculation miss rate.
For example, $\eta = 1.35$ asserts that the policy is expected to be profitable
whenever fewer than $1/1.35 \approx 74\%$ of admitted speculations go unused.

\section{Tier Placement}

When both CPU and GPU tiers are available, the system assigns each admitted prefix
to a tier based on an adjusted benefit that adds a GPU bonus $g > 0$ reflecting
the lower serving latency of GPU-resident KV states:
\[
    B_{\text{gpu}}(p) = B(p) + g
\]
The GPU tier incurs a higher precompute cost $C_{\text{gpu}}(p) > C_{\text{cpu}}(p)$
due to device transfer overhead.
A prefix is placed in the GPU tier if and only if
\[
    V_{\text{gpu}}(p) = B_{\text{gpu}}(p) - C_{\text{gpu}}(p) > V_{\text{cpu}}(p)
\]
Otherwise it is placed in the CPU tier.
